# aiffel-project

AI Agent를 만드는 과정을 정리한 저장소입니다.

## AI Agent를 만드는 방법

1. **목표와 역할 정의**
   Agent가 무엇을 해결할지, 어떤 입력을 받고 어떤 결과를 내야 하는지 명확히 정한다.
   - 해결하려는 문제와 사용자를 구체적으로 정의한다 (누가, 어떤 상황에서 사용하는가).
   - 성공 기준을 정한다 (예: "정확도 90% 이상", "5초 이내 응답").
   - Agent의 자율성 범위를 정한다 — 모든 행동을 자동 실행할지, 중요한 단계마다 사용자 확인을 받을지 결정한다.

2. **LLM 선택**
   Agent의 두뇌 역할을 할 언어 모델(Claude 등)을 선택하고, 시스템 프롬프트로 역할과 규칙을 정의한다.
   - 작업 난이도와 비용/속도를 고려해 모델 크기를 고른다 (간단한 작업은 소형 모델, 복잡한 추론은 대형 모델).
   - 시스템 프롬프트에 역할, 말투, 제약 조건(하지 말아야 할 것)을 명시한다.
   - 출력 형식을 구조화(JSON 등)해 이후 단계에서 파싱하기 쉽게 만든다.

3. **도구(Tools) 연결**
   웹 검색, 파일 읽기/쓰기, API 호출 등 Agent가 실제로 실행할 수 있는 도구를 정의하고 연결한다.
   - 각 도구의 입력/출력 스키마를 명확히 정의해 모델이 올바르게 호출하도록 돕는다.
   - 도구 설명은 구체적으로 작성한다 — 언제 사용해야 하는지 모델이 판단할 수 있어야 한다.
   - 위험한 작업(삭제, 결제, 외부 전송 등)은 실행 전 사용자 확인 절차를 둔다.

4. **메모리/컨텍스트 관리**
   이전 대화나 작업 결과를 기억하고 활용할 수 있도록 대화 이력, 벡터 검색(RAG) 등을 구성한다.
   - 단기 기억: 현재 세션의 대화 이력을 컨텍스트에 유지한다.
   - 장기 기억: 사용자 선호, 프로젝트 정보 등을 파일이나 벡터 DB에 저장해 세션 간에도 유지한다.
   - 컨텍스트가 길어질 경우 요약하거나 관련 정보만 검색(RAG)해 불러온다.

5. **계획 → 실행 → 관찰 루프**
   Agent가 스스로 다음 행동을 계획하고, 도구를 실행하고, 결과를 관찰해 다시 계획하는 반복 루프를 만든다.
   - 계획(Plan): 목표를 달성하기 위한 다음 행동을 모델이 스스로 결정한다.
   - 실행(Act): 선택한 도구를 호출해 실제 작업을 수행한다.
   - 관찰(Observe): 도구 실행 결과를 확인하고, 성공 여부에 따라 다음 계획을 세운다.
   - 무한 루프 방지를 위해 최대 반복 횟수나 종료 조건을 둔다.

6. **평가와 반복 개선**
   실제 사용 사례로 테스트하며 실패 케이스를 찾아 프롬프트/도구/로직을 개선한다.
   - 대표적인 테스트 케이스(정상/예외 상황 포함)를 만들어 반복 검증한다.
   - 실패 원인을 프롬프트 문제, 도구 문제, 로직 문제로 구분해 분석한다.
   - 사용자 피드백과 로그를 지속적으로 수집해 Agent의 행동을 개선한다.
